## What This Project Is

A portable e-ink display gadget (Raspberry Pi Pico WH + Waveshare 3.7" e-paper) that shows:
- **Idle mode:** Current time, date, weather + 6-hour forecast, rotating personal photo gallery
- **Now-playing mode:** Album art (4-level greyscale), track name, artist, album

All heavy processing (Spotify OAuth, image dithering, weather fetching) runs on a **Docker backend** on a home server. The Pico just polls `/status` and `/art/{id}` over local Wi-Fi.

See [README.md](README.md) for full architecture, hardware specs, and project phases.

## Current State of the Codebase

What exists right now (February 2026):
- `backend/renderer.py` — the core Pillow rendering engine for both screens. **This is the most complete and tested piece.**
- `backend/fonts/` — Geist font family (5 TTF files) used by the renderer
- `backend/photos/` — 57 pre-processed 225×185 greyscale BMPs for the photo rotation slot
- `tools/prepare_photos.py` — converts raw photos to e-ink-ready BMPs
- `tools/preview_renderer.py` — renders both screens locally and opens in macOS Preview
- `preview_art.py` — fetches a live BMP from the running server and opens it in Preview

What does NOT exist yet (planned):
- `backend/main.py` — FastAPI app (not scaffolded yet)
- `backend/spotify.py` — Spotify OAuth + polling
- `backend/weather.py` — Open-Meteo integration
- `backend/config.py` — env-based settings
- `backend/Dockerfile` + `docker-compose.yml`
- `firmware/` — MicroPython for the Pico (waiting for hardware)

## Key Technical Constants

These values are **tightly coupled** across files — changing one requires changing the other:

| Constant | Value | Where it lives |
|---|---|---|
| Canvas size | 480 × 280 px | `renderer.py` W, H |
| Photo slot size | 225 × 185 px | `renderer.py` PHOTO_W, PHOTO_H AND `tools/prepare_photos.py` PHOTO_W, PHOTO_H |
| Album art size | 280 × 280 px | `renderer.py` ART_W, ART_H |
| Greyscale levels | 0, 85, 170, 255 | `renderer.py` + `tools/prepare_photos.py` |
| Photo rotation | 10 minutes/photo | `renderer.py` PHOTO_ROTATE |
| Server address | `http://100.84.37.35:8765` | `preview_art.py` API_BASE |
| Server SSH alias | `cribbbserver` | Referenced in `tools/prepare_photos.py` scp comment |

## Display Specs

- **Resolution:** 480 × 280 pixels, landscape
- **Colour depth:** 4-level greyscale only (0=black, 85=dark grey, 170=light grey, 255=white)
- **Dithering:** Floyd-Steinberg error diffusion (applied to all output BMPs)
- **Full refresh:** 3 seconds (ghosting cleared)
- **Partial refresh:** ~0.3 seconds (no flash)

## /status JSON Shape

The backend's `/status` endpoint is the Pico's single source of truth. The renderer expects this structure:

```python
{
    "time": "14:32",           # HH:MM string
    "date": "Thu 19 Feb",      # formatted date string
    "weather": {
        "temp_c": 8,           # integer or float
        "condition": "drizzle", # see ICONS dict in renderer.py for valid values
        "forecast": [          # list of up to 6 hourly slots
            {
                "hour": "15:00",
                "temp_c": 7,
                "condition": "rain"
            },
            # ... up to 6 total
        ]
    },
    "spotify": {
        "is_playing": True,
        "track": "Pyramid Song",
        "artist": "Radiohead",
        "album": "Amnesiac",
        "track_id": "abc123",
        "art_url": "/art/abc123"
    }
}
```

Valid weather condition strings (map to icons in renderer.py):
`sunny`, `partly_cloudy`, `cloudy`, `overcast`, `fog`, `drizzle`, `rain`, `snow`, `showers`, `thunderstorm`

## Common Tasks

### Preview both screens locally
```bash
~/miniforge3/bin/python3 tools/preview_renderer.py
```
Outputs to `/tmp/preview_idle.png` and `/tmp/preview_now_playing.png`, auto-opens in Preview.

### Rebuild photo library
```bash
# 1. Drop raw images into ~/Desktop/rendered eink pics/
# 2. Run:
~/miniforge3/bin/python3 tools/prepare_photos.py
# 3. SCP results to server:
scp backend/photos/*.bmp cribbbserver:~/pico-eink-display/backend/photos/
```

### Fetch live album art from running server
```bash
~/miniforge3/bin/python3 preview_art.py
# Requires server running at http://100.84.37.35:8765
```
